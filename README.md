# official-wc-hub

这是一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，主要目的是提供一个集中存放、管理和展示静态 HTML 资源的地方。

## 项目简介

本仓库收纳了多个独立开发的 HTML 页面，涵盖不同的功能或展示用途。每个页面均为自包含的 HTML 文件，不依赖外部数据库或后端服务，适合直接通过 GitHub Pages 或其他静态托管服务部署。

## 目录说明

- `pages/` — 存放所有归档的 HTML 页面文件，每个文件独立命名，便于直接访问。
- `assets/` — 可选的公共资源目录，用于存放页面共用的样式、脚本或图片文件。
- `docs/` — 如果启用 GitHub Pages，该目录可作为发布根目录（可根据需要调整）。

仓库根目录下可能包含少量配置文件（如 `index.html` 或 `CNAME`），用于优化展示效果。

## 页面归档说明

- 每个 HTML 页面均为独立归档，内容完整，可直接在浏览器中打开运行。
- 页面之间没有强制关联，可按需单独引用或部署。
- 归档的页面可能包含本地化资源引用，建议使用相对路径以确保在子目录下正常加载。

## 维护说明

- 本仓库定期更新，新增页面会以独立文件形式加入 `pages/` 目录。
- 欢迎提交 Issue 或 Pull Request 来提议新增页面或修复现有问题。
- 请勿在页面中包含恶意脚本、跟踪代码或违反平台政策的内容。
- 如需贡献，请确保 HTML 文件符合基本语法规范，并尽量保持文件结构清晰。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/official-wc-hub.git
   ```
2. 直接打开 `pages/` 下的任意 HTML 文件，或通过本地服务器预览。
3. 若启用 GitHub Pages，可配置发布源为 `main` 分支的根目录或 `docs/` 目录，随后通过对应 URL 访问页面。
